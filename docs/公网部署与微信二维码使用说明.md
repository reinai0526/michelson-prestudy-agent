# 公网部署与微信二维码使用说明

目标：让学生不在同一个 Wi-Fi 下，也能通过微信扫码或打开网址使用平台。

## 推荐方案：部署到公网平台

二维码本质上只是一个网址。要让学生随时扫码可用，需要先获得一个公网 HTTPS 网址，例如：

```text
https://michelson-prestudy-agent.onrender.com
```

项目已经适配为公网部署版本：

- 前端页面和 DeepSeek 问答接口由同一个 Node 服务提供；
- DeepSeek API Key 只放在服务器环境变量中，不会出现在学生浏览器里；
- 云平台自动分配端口时，服务会自动读取 `PORT`。

## Render 部署步骤

1. 将本项目上传到 GitHub 仓库。

2. 打开 Render，新建 Web Service，并连接这个 GitHub 仓库。

3. Render 会读取项目中的 `render.yaml`，也可以手动填写：

```text
Build Command: npm ci && npm run build
Start Command: npm start
```

4. 在 Render 的 Environment Variables 中添加：

```text
NODE_ENV=production
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-v4-flash
DEEPSEEK_API_KEY=你的真实 DeepSeek Key
```

5. 部署完成后，Render 会给出一个 HTTPS 网址。学生打开这个网址即可使用。

## 生成微信二维码

拿到公网 HTTPS 网址后，用任意二维码工具生成二维码即可，例如：

```text
https://cli.im
```

也可以把网址发到微信里，长按链接或使用微信自带工具生成/分享二维码。

学生扫码打开后，看到的是同一个智能体平台。

## 注意事项

- 不要把 `.env.local` 发给学生，也不要把 DeepSeek Key 写进前端页面。
- 学生人数较多时，DeepSeek API 调用会产生费用，请控制测试人数和提问频率。
- 免费云平台可能会休眠，第一次打开可能需要等待几十秒。
- 若学校网络限制外部平台访问，建议改用学校服务器或腾讯云/阿里云轻量服务器部署。

## 临时测试方案

如果只是短期测试，也可以使用公网隧道工具把本机服务临时暴露到公网，但需要你的 Mac 一直开机，终端不能关闭。课堂正式测试更推荐部署到公网平台。
